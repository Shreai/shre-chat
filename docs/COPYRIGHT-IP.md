# Copyright and IP Policy

> Template only. This is an operating document, not legal advice.
> Ownership rules can differ by contract, country, and work-for-hire treatment.

## Purpose

This document defines how content, code, designs, and deliverables are owned and licensed.

## Default Rules

- client-funded deliverables should specify ownership in the project contract
- internal work should specify whether Nirlab, Shre.ai, or another entity owns the output
- source code, UI assets, copy, and documentation should all have explicit ownership treatment
- third-party libraries and media must be tracked with their licenses
- AI-generated content must be reviewed for rights, training-data concerns, and commercial suitability before use

## Contract Fields to Define

- owner of source code
- owner of design assets
- owner of copy and documentation
- license granted to the customer, if ownership is not transferred
- reuse rights for platform components
- attribution requirements
- open-source notices and obligations
- trademark usage rules

## Operating Rules

- do not assume work is automatically transferable without contract language
- keep reusable platform modules separate from client-owned deliverables
- track every external asset in the connector or dependency catalog, as applicable
- preserve evidence of permission for stock media, fonts, and third-party content

## Review Triggers

- client handoff
- white-label launch
- reuse of custom code across projects
- use of third-party media or brand assets
- trademark, logo, or naming questions
