# Threat Model

## Purpose

Record the main threats before building a product or major feature.

## Checklist

- assets
- trust boundaries
- attackers
- likely abuse paths
- data exposure risks
- auth risks
- connector risks
- rollback risks
- network and edge risks
- DDoS risks
- injection risks
- crawling/scraping risks
- sniffing/listening risks

## Rules

- update when auth, API, or data flows change
- review before launch
- tie mitigations to concrete controls
- pair each major threat with a documented control in security or operations docs
