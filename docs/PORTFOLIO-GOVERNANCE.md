# Portfolio Governance

This document controls how major products and programs are prioritized, funded, and reviewed.
It is the executive layer above a single project SOW.

## Purpose

- choose what gets funded first
- define success at the product or program level
- keep budget and roadmap decisions visible
- give leadership a simple operating view

## Core Fields

- portfolio item
- owner
- objective
- status
- funding level
- priority
- target date
- risk level
- metric or OKR
- dependencies

## Rules

- every strategic product should have a portfolio entry
- every portfolio item should have a clear owner
- every item should have a metric that shows whether it is working
- every item should have a review date
- if a bet becomes a major risk, escalate it through governance review

## Review Cadence

- weekly: active portfolio items
- monthly: portfolio reprioritization
- quarterly: budget and roadmap reset

## Output

- prioritized list of bets
- paused items
- funded items
- executive notes
- risks and blockers

## Rule

If a project matters to the business, it should appear in portfolio governance.
